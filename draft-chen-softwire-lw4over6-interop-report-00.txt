Abstract

   The document describes the test result of a network employing
   Lightweight 4over6, which is a stateful IPv4-over-IPv6 tunnel
   transition mechanism with the feature of address sharing.

1.  Introduction

   We carried out an interoperation test of lightweight 4over6 mechanism
   [I-D.cui-softwire-b4-translated-ds-lite] from Oct. 22, 2012 to
   Oct.26,2012 at Tsinghua University, Beijing, China.  We tested the
   functional features of the system, the capacity of supporting IPv4
   applications and interoperation among lwB4s and lwAFTRs that
   implemented by different venders and operators.

   The test mainly focused on checking whether the multiple
   implementations of lightweight 4over6 mechanism could work correctly,
   and on the interoperation between lwB4 and lwAFTR from different
   vendors and operators as well.

   According to the test, we can see that the lightweight 4over6
   mechanism works well and we can figure out the user experience of
   different applications and the impact on a typical Internet user.

5.  Interoperation Test Results

   We tested functions of the implementation of the lwB4 and lwAFTR on
   both control plane and data plane.

   The function test cases include DHCP and PCP basic process, IPv4
   address renewing and releasing, and other must-be features like the
   NAPT function of lwB4, and IPv4-over-IPv6 tunneling of both lwB4 and
   lwAFTR.

   For DHCP provisioning test, different scenarios have been tested
   including integrated DHCPv4 server and standalone DHCPv4 server.  In
   the mode of integrated DHCPv4, DHCPv4-over-v6 server should be
   supported by lwAFTR to determine the bindings between IPv6 address,
   IPv4 address and port-set dynamically.  In the mode of standalone
   DHCPv4, lwAFTR can either support DHCPv4 relay to retrieve the
   binding information on path, or support additional synchronization
   mechanism with DHCPv4 server to setup the binding table.

   The application test cases include the most widely used applications
   in China, including text website, video stream website, webmail,
   instant messaging and web dynamic content.  We also downloaded
   resources by several of download tools.  They all work correctly.
   There is not obvious difference in speed between the host in LAFT6
   system and the host outside it.  Other applications, like ping and
   tracert, are also tested.  According to the test, we know that the
   lightweight 4over6 mechanism supports almost all kinds of traditional
   IPv4 applications,and there is no obvious difference in speed between
   the host in lw4over6 system and the host outside it.

   In addition, the test results of various combinations of lwB4s and
   lwAFTRs that provided by the same or different venders show that the
   implementations are well done and have a good feature of
   interoperation.

5.3.  Conclusions

   The results of function inner test (i.e. lwB4 A communicates with
   lwB4 A where A stands for the same number) show that the1 equipments
   from the same provider can perform a good collaboration on IPv4
   provisioning.

   The results of function interop test (i.e. lwB4 A communicates with
   lwB4 B where A and B stand for different numbers) show that
   equipments from different providers can co-operate well on IPv4
   provisioning.



Chen, et al.              Expires May 11, 2013                 [Page 24]

Internet-Draft           Lw4over6 Interop Report           November 2012


   The results of both application inner test and interop test show that
   equipments from the same or different providers can provide a good
   support for the IPv4 applications.
